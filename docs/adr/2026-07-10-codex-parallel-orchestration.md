# ADR 2026-07-10 — Blocking codex-worker: Codex units as first-class Workflow nodes

**Status:** accepted
**Context.** The `codex:codex-rescue` forwarder is fire-and-forget: long Codex tasks background inside the companion and `agent()` resolves on a job-handle string, breaking `parallel()`/`pipeline()` barriers. Production evidence (clusterbid spec 037): hand-rolled Bash watchers misfiring, three silent ~2h hangs (root cause: Codex invoking a collaboration/wait tool that blocks forever headless), forced wave barriers, cwd-dependent sandbox roots, and a three-part model-pin trap (CLI ≥0.144 + stale app-servers + shadowed installs). Same night, this repo's own 012 build reproduced the failure: a write-unit dispatched via the forwarder returned handle-shaped output and degraded to Claude (pilot-ledger row 5).

**Decision.** (1) Canonical `core-rules/agents/codex-worker.md` — a cheap Claude driver with a BLOCKING contract: explicit `--background` launch, top-level jobId capture, foreground 30s poll chunks (never run_in_background), unattended preamble (no collaboration/wait/ask tools), no-session-id-in-5-min → cancel + one retry a tier lower, 15-min log-mtime stall → cancel + one annotated relaunch → FAILURE, result + diff-stat + receipt (attempts, downgrades, ids). Cancel is not proof of termination — post-cancel status is checked and noted. (2) `codex-fanout.wf.js` — mixed-leg fan-out with required per-unit effort (011 contract), config-knob concurrency cap (`trellis.config.json.codex_fanout.concurrency`, default 4; Claude legs consume no slot), topologically-ordered waves honoring `dependsOn`, branch-based isolation for conflicting units (commit-to-`unit/<name>`, verify diffs the branch — worktree-per-stage isolation severs generate from verify and is rejected), anchored receipt-block STATUS parsing, job-handle leak guard with cancel-on-leak. (3) All Workflow dispatch paths migrate to the worker (`codex-executor`, `verify-panel`, `fleet-audit-remediation`); the rescue path is demoted to interactive-only. (4) Speed doctrine (`references/speed-doctrine.md`): race-the-legs, cross-harness pipelined verify, warm-thread pool, primer-fed dispatch, streaming merges, ultra-as-node (locked on 011 D4a) — speed from topology, not effort. (5) Agent defs inherit fleet-wide via `.claude/agents/` symlinks; `core-rules/agents/` ships to the public mirror (capability-gated, inert without the plugin).

**Consequences.** Codex agents compose in pipelines/judge-panels/verify stages identically to Claude agents — the operator's two subscriptions parallelize as one fleet. Stalls become bounded failures with receipts instead of silent hangs. Cost: a polling driver per Codex unit (~free Sonnet courier), a stricter dispatch contract, and a per-checkout warm-thread state file. Known follow-ups: wave-drain barrier under-delivers the no-barrier pipelined-verify ideal (design note); the stall suite validates the contract's observable artifacts + pinned constants, not the live agent prose.
