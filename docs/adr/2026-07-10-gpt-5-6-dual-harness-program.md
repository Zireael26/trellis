# ADR 2026-07-10 — GPT-5.6 adoption program: effort re-ground, follow-ups convention, Codex parallel orchestration

**Status:** accepted
**Context.** OpenAI shipped GPT-5.6 (Sol/Terra/Luna) publicly 2026-07-09: new `max` reasoning effort and `ultra` multi-subagent mode, ~half Fable-5 token price on Sol, and a system card documenting increased agentic-coding overreach (most pronounced at highest effort under persistence-heavy prompts). Three Trellis doctrine surfaces carried now-false "no max for Codex" ceiling claims; recipes hardcoded blanket xhigh; the fire-and-forget Codex dispatch path broke workflow barriers in production (clusterbid spec 037). Operator directive: gpt-5.6-sol as the Codex model, medium→xhigh operating band, max/ultra as exception tiers, maximum dual-harness parallelization, follow-up suggestions captured automatically.

**Decision.** Three-spec program, one intake:
- **Spec 011** — effort ladder v2 (explicit-effort-or-error, exception tiers receipt-justified, ultra locked out of recipes until telemetry + ×4 cap + instrumented run), stale ceiling claims killed on active surfaces (historical artifacts exempt), safety clauses landed in runnable prompt builders (not just prose), §2 strength table re-ground evidence-gated (≥2 independent non-OpenAI evals, expiry 2026-08-15). Codex adversarial review (Sol xhigh) verdict RETHINK folded as r2.
- **Spec 012** — follow-ups convention: automatic prioritized follow-ups at spec/PR/DoD boundaries, zero-extra-token derivation, spec-section + root-ledger capture, non-blocking stop-verify warn, route-by-size disposition.
- **Spec 013** — blocking `codex-worker` canonical agent + `codex-fanout` recipe (adopts the clusterbid handoff: 30s poll / 15-min stall / one retry / FAILURE report; unattended preamble after the collaboration-tool stall root-cause), six-pattern dual-harness speed doctrine (race-the-legs, cross-harness pipelined verify, warm-thread pool, primer-fed dispatch, streaming merges, ultra-as-node) — speed from topology, not effort.
Companion doctrine flip, same evidence base: dynamic workflows preferred over named teammates for fan-out (teammates never auto-terminate; teammate panes spawn with hooks dead — node absent from PATH).

**Consequences.** Codex units become first-class workflow nodes with pattern parity to Claude agents; effort mis-tiering (blanket xhigh) ends; exception tiers are reachable but never default; executor honesty is contract-enforced where the system card says risk concentrates; the strength-routing table stays honest through the launch window instead of absorbing launch-day claims; every completion boundary now emits durable, prioritized follow-ups. Cost: three PRs of doctrine/recipe churn, one new canonical agent + recipe to maintain, and a stricter dispatch contract (explicit effort everywhere) that all future recipes must honor.
